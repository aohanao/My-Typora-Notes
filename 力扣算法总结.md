# 力扣算法通用解法

## 1. 回溯算法框架

回溯的本质是：**做选择 → 递归探索 → 撤销选择**，核心是通过「试错」的方式遍历所有可能的解空间。

我先给你一个通用的伪代码模板，再拆解每个部分的作用：

```python
def backtrack(参数):
    # 1. 终止条件：找到一个合法解，记录结果
    if 满足终止条件:
        把当前路径加入结果集
        return
    
    # 2. 遍历所有可选的「选择」
    for 选择 in 选择列表:
        # 剪枝：排除不合法/无效的选择（可选，但能大幅提升效率）
        if 选择不合法:
            continue/break
        
        # 3. 做出选择：把当前选择加入路径
        路径.append(选择)
        
        # 4. 递归探索：进入下一层决策树
        backtrack(更新后的参数)
        
        # 5. 撤销选择：回溯，回到上一步（关键！）
        路径.pop()

# 主函数：初始化结果集和路径，调用回溯函数
def main(问题参数):
    结果集 = []
    路径 = []
    backtrack(初始参数)
    return 结果集
```

### 模板核心拆解（5 个关键部分）

我用「人话 + 对应组合总和案例」帮你理解每个部分的作用：

| 模板部分 |              核心作用              |             组合总和中的对应实现             |
| :------: | :--------------------------------: | :------------------------------------------: |
| 终止条件 |  划定递归的边界，找到有效解时记录  | `if remain == 0: result.append(path.copy())` |
| 遍历选择 |      遍历当前层所有可做的选择      |   `for i in range(start, len(candidates))`   |
|   剪枝   |    提前排除无效选择，避免走弯路    |    `if num > remain: break`（排序后剪枝）    |
|  做选择  | 将当前选择加入「路径」（记录已选） |              `path.append(num)`              |
| 递归探索 |       带着新的状态进入下一层       |      `backtrack(remain - num, path, i)`      |
| 撤销选择 |      回到上一层，尝试其他选择      |                 `path.pop()`                 |

### 模板的灵活适配（不同题型的调整点）

你只需要根据题目的**核心约束**，调整模板中的「选择列表」和「参数」即可，我用常见题型举例：

#### 1. 组合类问题（如组合总和、组合总和 II、组合）

- **核心约束**：元素可重复选 / 不可重复选、组合不重复（[2,3] 和 [3,2] 算同一个）

- **调整点**：

  - 用 `start` 参数控制「选择列表的起始位置」（避免重复组合）；
  - 可重复选：`backtrack(..., i)`（下一层仍从当前索引开始）；
  - 不可重复选：`backtrack(..., i+1)`（下一层从下一个索引开始）。


#### 2. 排列类问题（如全排列、全排列 II）

- **核心约束**：元素可重复 / 不可重复、排列要去重

- **调整点**：

  - 不用 `start` 参数，而是用「used 数组」标记已选元素；
  - 去重：排序后跳过重复元素 + 检查 `used` 数组。


#### 3. 子集类问题（如子集、子集 II）

- **核心约束**：收集所有子集、子集不重复

- **调整点**：

  - 终止条件不是「凑够目标」，而是「遍历完所有元素」（或每一步都把路径加入结果）；
  - 用 `start` 参数控制起始位置，避免重复子集。


#### 4. 棋盘类问题（如 N 皇后、解数独）

- **核心约束**：满足棋盘规则（如皇后不冲突）
- **调整点**：
  - 「选择列表」是当前行的所有列；
  - 「剪枝」是检查当前位置是否符合规则（如皇后是否冲突）；
  - 终止条件是「遍历完所有行」。
